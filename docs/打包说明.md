# 打包成可执行应用（无需安装 Python）

把桌面宠物打包成 macOS 的 `.app` 或 Windows 的 `.exe`，用户双击即可启动，无需安装 Python 或任何依赖。

## 重要前提

**跨平台打包：必须在各自平台上打包。**
- macOS 的 `.app` 只能在 **Mac** 上打包
- Windows 的 `.exe` 只能在 **Windows** 上打包
- 没有工具能在 Mac 上产出 Windows 可执行文件，反之亦然

## macOS 打包

### 一键打包
```bash
bash scripts/build_macos.sh
```
产出 `dist/DesktopCat.app`（约 64MB），双击即可启动。

### 手动分步
```bash
# 1. 打包
python3 -m PyInstaller desktop-cat.spec --noconfirm
# 2. 瘦身（删无用 Qt 模块，从 587MB 降到 64MB）
python3 scripts/strip_app.py
# 3. 重新签名（瘦身删了文件后签名失效）
codesign --force --deep --sign - dist/DesktopCat.app
```

### 使用 / 分发
- **本地使用**：把 `dist/DesktopCat.app` 拖到「应用程序」文件夹，像普通 App 一样从 Launchpad 启动
- **分发**：可压缩成 zip 发给别人，解压后双击即可（首次打开可能要在「系统设置 → 隐私与安全性」点「仍要打开」，因为用的是自签名）

## Windows 10/11 打包

### 前提
在 Windows 机器上，装好 Python 3.10+ 和项目依赖：
```cmd
pip install -r requirements.txt
pip install pyinstaller
```

### 一键打包
```cmd
scripts\build_windows.bat
```
产出 `dist\DesktopCat\DesktopCat.exe`，双击即可启动。

## Windows 7 SP1 x64 打包

Windows 7 专用包使用 Python 3.8 + PySide2/Qt5。可以在 Windows 11
构建，但必须使用独立虚拟环境，不能混入 PySide6。

### 安装与准备

```cmd
py install 3.8
py -3.8 -m venv .venv38
.venv38\Scripts\activate
python --version
python -m pip install --upgrade pip
python -m pip install -r requirements-win7.txt
```

`python --version` 必须显示 Python 3.8.x。

### 构建

```cmd
scripts\build_windows7.bat
```

产物仍位于 `dist\DesktopCat`。分发时压缩整个目录：

```powershell
Compress-Archive -Path dist\DesktopCat -DestinationPath DesktopCat-windows7.zip -Force
```

必须在安装完整更新的 Windows 7 SP1 x64 真机上验证。包内不应出现
`PySide6`、`Qt6Core.dll` 或 `python311.dll`。不要从其他系统复制
`api-ms-*.dll` 到 Windows 7。

### 分发
把整个 `dist\DesktopCat` 文件夹压缩成 zip 发给别人，解压后双击 `DesktopCat.exe`。

## 体积优化说明

PyInstaller 默认会把 Qt Python 绑定（PySide6 或 PySide2）中未使用的模块打进去。

通过 `scripts/strip_app.py` 删除无用模块，降到约 64MB：
- 删 Qt3D / QtWebEngine（216MB）/ QtQuick / QtQML
- 删 QtMultimedia / QtCharts / QtPdf / QtNetwork 等
- 删 Qt 开发工具（Assistant / Linguist / Designer）
- 只保留 QtCore / QtGui / QtWidgets + 基本显示插件

如果运行时报 `ModuleNotFoundError`，说明 `desktop-cat.spec` 的 `excludes` 排除了过多模块，把对应模块从 `excludes` 列表移除即可。

## 自定义图标

默认无图标。要加图标：
1. 准备一个 `.icns`（macOS）或 `.ico`（Windows）图标文件
2. macOS：在 `desktop-cat.spec` 的 `BUNDLE(icon='your_icon.icns')` 设置
3. Windows：在 `desktop-cat.spec` 的 `EXE(icon='your_icon.ico')` 设置

## 修改打包后的应用名/版本

在 `desktop-cat.spec` 末尾的 `BUNDLE` / `info_plist` 里改：
- `name='DesktopCat.app'` → 应用名
- `CFBundleVersion` / `CFBundleShortVersionString` → 版本号
- `CFBundleDisplayName` → 显示名（中文）

## 常见问题

**Q: 打包后启动闪退？**
A: 直接运行 `dist/DesktopCat.app/Contents/MacOS/DesktopCat` 看错误输出，通常是缺模块——调整 spec 的 excludes/hiddenimports。

**Q: macOS 提示"无法打开，因为无法验证开发者"？**
A: 这是自签名的正常提示。到「系统设置 → 隐私与安全性」点「仍要打开」，或 `xattr -cr dist/DesktopCat.app` 清除隔离属性。

**Q: 体积还是太大？**
A: 确认跑了 `strip_app.py`。还可以用 UPX 压缩（spec 里 `upx=True` 已开，需要本机装 UPX）。
